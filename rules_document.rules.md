# RULES – 제목 A/B 테스트 예측기

## MVP 원칙
- 최소 기능 구현: CTR 예측 + 추천
- 불필요한 기능 배제, 빠른 배포

## 디자인 원칙
- 다크모드 기본
- 현대적이고 심플한 레이아웃
- 로고/파비콘 필수 적용
- 반응형 지원

## 광고 규칙
- 자동광고: 좌/우 사이드바
- 수동광고: 상단/중간/하단
- 광고가 콘텐츠를 방해하지 않도록 배치

## API 규칙
- Google Gemini API 사용
- 사용자가 직접 API KEY 입력
- API KEY 안내 팝업 필수
  - 초등학생도 이해할 수 있도록 설명
  - 공식 문서 링크 포함
  - ESC 키로 빠른 닫기 가능

## 배포 규칙
- Vercel 배포
- SEO 최소 적용 (title/meta/og 태그)

## 확장성
- 추후 CTR 예측 외 기능 추가 가능 (예: 설명문 최적화, 해시태그 추천)
- 컴포넌트 기반 설계 권장
